Arquitetura Técnica & Documentação de Engenharia

Um mergulho profundo, vírgula por vírgula, na estrutura que compõe o ecossistema NicheFinder Guru. De requisições API geoespaciais e coleta de dados na web à renderização de PDF sob demanda e automação de conexões do WhatsApp.

Arquitetura do Sistema

O NicheFinder Guru não é um sistema monolítico (tudo num lugar só). É um ecossistema distribuído projetado para contornar limitações de rede, bloqueios do WhatsApp e pesadas cargas de geração de arquivos. A arquitetura divide a Inteligência da Execução.

// O Fluxo Fundamental da Aplicação

1. [Frontend React] Dispara busca de locais no Google via Funções Auxiliares.

2. [Coletor Web] Audita o site (ou a ausência dele) e insere no Banco PostgreSQL (Supabase).

3. [Inteligência Groq] Analisa a falha (ex: "Sem site no mapa") e cria a argumentação de venda.

4. [Serviço PDF] Frontend envia dados via rede para a porta local `3001` que gera o PDF.

5. [Robô WhatsApp] Processo independente consome a fila do banco e dispara o PDF e o texto de forma nativa.

# Stack Tecnológica

Interface Ocular (Frontend)

  • React 18 + TypeScript
  • Vite (Empacotador Rápido)
  • Tailwind CSS v3 (Estilização Útil e Bruta)
  • Zustand / React Query (Gerenciamento de Estado)
  • Lucide (Sistema de Ícones)

Estrutura de Dados (Backend)

  • Supabase (PostgreSQL 15)
  • Supabase Auth (Políticas de Segurança e Acesso)
  • Node.js (Focos e Trabalhadores Independentes)
  • Puppeteer (Gerador Chrome Invisível)

Motor Radar B2B (Mineração)

A fonte primária de dados. Localizado em /src/pages/Radar.tsx, este motor é o responsável pela captação cirúrgica de contatos de empresas locais no Brasil inteiro.

Vírgula por vírgula: O Ciclo de Mapeamento

  • 1. Parâmetros de Entrada: O usuário insere o nicho (ex: "Clínica de Estética") e a localidade. O sistema permite pesquisar múltiplas cidades separadas por vírgula de uma só vez.
  • 2. Requisição Intermediária: Para não vazar a sua chave do Google Maps direto no navegador, o disparo ocorre através de uma Função de Borda (Edge Function) segura no Supabase.
  • 3. Coleta de Informações: A API devolve o Nome, Avaliação, Quantidade de Estrelas, Endereço Completo e o mais crítico de todos: O Endereço do Site.
  • 4. A Lógica "Mina de Ouro" (Sem Site): O radar instantaneamente detecta e marca em vermelho as empresas que o sistema Google retornou campo de "site vazio". Essa é a empresa que fatura no espaço físico, mas é invisível aos clientes digitais.
  • 5. Filtro de Lista Negra: Antes de mostrar o resultado na tela, os dados passam por uma validação que exclui do sistema alvos indesejados como "Prefeituras", "Sebrae" ou outras "Agências de Marketing" locais.

Auditor Técnico

Mas e as empresas que já possuem um site? O sistema não as ignora. Ele procura as falhas de implantação delas. O script siteAuditor.ts assume o controle.

// Lógica assíncrona do Auditor Técnico
const resultadoAuditoria = await executarEscaneamentoProfundo(novo_site);
if (resultadoAuditoria.ssl_seguro === false) {
  marcarAlvoComo('VULNERAVEL_SEM_HTTPS');
}
if (resultadoAuditoria.velocidadeDaPagina < 50) {
  marcarAlvoComo('LENTIDAO_CRITICA');
}

O auditor faz verificações profundas direto ao protocolo do site para checar certificados de segurança válidos. Em seguida, dispara uma verificação conectada à tecnologia do Google PageSpeed, testando exclusivamente a velocidade no Celular e coletando quanto tempo em segundos o site demora pra abrir.

Classificação Térmica (O Score)

Se nós fôssemos apenas listar as empresas em ordem alfabética da cidade, seríamos as "Páginas Amarelas" antigas. O NicheFinder usa o algoritmo Classificação Térmica inteligente para jogar para o topo da sua tabela quem está sangrando dinheiro e implorando por uma venda.

A Hierarquia de Dor (Quente = Mais Vermelho na Interface)

  1. STATUS: SEM_SITE (Força: 100) - O alvo absoluto de ataque. Lucro perdido que você resolve facilmente.
  2. STATUS: SEM_CADEADO (SSL) (Força: 80) - O site marca "Inseguro" no celular dos clientes.
  3. STATUS: LENTO_NO_CELULAR (Força: 60) - Uma nota de velocidade menor que 50 no sistema do Google.
  4. STATUS: SAUDÁVEL (Força: 10) - A empresa tem presença excelente. Fica invisível no fim da sua lista para não tomar seu tempo.

Integrações de Inteligência Artificial

Nós conectamos diretamente o motor brutal da Groq no script aiService.ts. Trata-se do processamento mais rápido mundial na atual geração textual, garantindo que o robô não engasgue sob demanda massiva brasileira.

Comando Interno (Exemplo para problema "Falha no Celular/Lento")

"Você é um engenheiro sênior persuasivo. A mecânica ${nomeDaEmpresa} da zona leste tem ótimas resenhas físicas do público local (${nota} estrelas). Mas eles possuem site com lentidão absurda, rodando sem cadeado no celular e ninguém entra no link que puseram lá no Maps. Crie uma mensagem curta de três frases para chamar o dono no WhatsApp de forma orgânica, dizendo que um vizinho deles notou que o painel online está falhando no Chrome."

Esta resposta da IA nunca é padronizada ou salva em pedra. Como cada contato sofre uma análise ao vivo, cada proprietário vai ler algo que parece ter sido digitado e observado exatamente e unicamente para ele naquela manhã.

O Gerador de PDF VIP: Motor Puppeteer

Talvez este seja o maior hack de arquitetura que aplicamos. O navegador padrão trava e chora pra criar PDF customizado (ele exige clique manual do usuário para "Salvar Página Web"). Nossa solução: levantar uma minúscula fábrica invisível separada usando NodeJS! Em vez do Google Chrome fazer força, nosso servidor miniatura faz sozinho.

Como Nasce uma Proposta VIP de R$5.000:

1. Empacotar Envio
O Frontend agrupa cores perfeitas, textos e um layout incrivelmente rico projetado em Tailwind React. Ele transforma isso tudo numa grande string de texto nativa e joga pelo caminho de rede via POST.
2. Manipular Pelo Fantasma
O pdf-engine.js usa nossa ferramenta Puppeteer. Ele abre de fundo um Chrome Fantasma (que você nem vê), injeta ali todo aquele amontoado de texto visual recebido e aguarda todas as imagens e fontes externas instalarem.
3. Imprimir Memória e Retornar
Sem salvar nenhum PDF na sua máquina, nosso fantasma formata o documento completo perfeitamente na proporção da folha A4 e manda via código a tradução em base de texto (`base64`). Sem lixo oculto no HD.

O Robô Autônomo WhatsApp (Standalone)

Tentar invadir as portas do navegador com disparos em massa geraria imediato bloqueio por robôs de patrulha do Facebook Meta (O famoso BAN na conta WA). Fomos forçados a programar um soldado à parte, o wa-bot.js baseado no protocolo oficial direto via soquetes de comunicação nativos da máquina!

Engenharia Anti-Ban e o Fio Condutor (Fila / Queue)

Nossa automação nunca dá as mãos pra plataforma de clique principal. O Robô fica calado conectado nos trilhos do Servidor Banco de Dados `whatsapp_queue`. Você aciona "Começar" no painel gerencial, e os comandos viram pequenas bandeiras.

Passo a Passo da Operação Segura Autônoma:

  • → Robô acorda na sombra, pede informações pela bandeira 1 na Fila.
  • → Pergunta secretamente ao WA Meta "Esse número existe lá?" usando protocolo seguro.
  • → Confirma e aciona o status digital "Digitando..." e paralisa na quantidade de tempo correta dependendo se a argumentação Groq foi curta ou longa (humanização técnica impecável).
  • → Evita imagens com baixa qualidade e aciona disparo oficial do PDF mágico. Segundos depois manda o texto persuasivo do Groq por baixo.
  • → A Mágica Anti-Robôs Meta: Força uma sesta técnica em um número de segundos sempre completamente modificado no intervalo entre 45 e 120 antes de ousar enxergar as próximas opções.
  • → Notifica no servidor a tag "ENVIADO!". O seu sistema reflete alterando a cor ao vivo de vermelho pra ver.

Interface de Usuário Local

Todo o ambiente da operação foi blindado sob um visual pesado Brutalista, tons negros profundos com letreiros chamativos pontuais. Tiramos todas as distrações para forçar atenção máxima 100% onde o trabalho impera. Sem perda de tempo, controle de interface sempre via menu flutuante fixado pela vertical esquerda.

A. Central Radar Diária

Ponto oficial de inicialização territorial. Possui apenas as entradas diretas de Município e o Ramo Comercial que será tratorado. Assim que inicia via cliques visuais, simula esteticamente processos de caça nativa fluida de dados. Revela os blocos avermelhados pontuais alertando os valiosos perfis "Sem Conexão Online/Sem Site".

B. Setor Comercial / Linha Viva (CRM)

Tabela de grande capacidade desenhada pro processamento paginado com limite e scroll instantâneo infinito ao vivo. Os emblemas e notas ordenadas sob hierarquia técnica são espelhados imediatamente no visual das caixas, facilitando você clicar exatamente na maior carência e bater num clique veloz disparando uma Auditoria e um VIP PDF gerado antes de respirar.

C. Base Operacional de Tiro (Status Realtime)

Quadro vivo do seu robô independente de ataques no Whatsapp Node. Totalmente passivo, este menu atua puramente pela subscrição instantânea do WebSockets nativo via Supabase para alterar caixas textuais. O ato de pressionar os ícones geram chamadas no back-end alterando as engrenagens silenciosamente. Exibe estatísticas vitais visuais dos processos do cérebro "em andamento", "falhados" ou "vencidos na hora".
