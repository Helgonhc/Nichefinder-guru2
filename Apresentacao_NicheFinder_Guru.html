O Guia Definitivo e Absoluto.

Você pediu para explicarmos TUDO. Este documento detalha, botão por botão, bit por bit, cada módulo, regra de negócio, limite e inteligência contida no NicheFinder Guru.

Módulo 1: O Radar (Data Mining)

1.1. Buscas de Múltiplas Cidades Simultâneas

A barra de busca do Radar não pesquisa apenas um local por vez. Ela foi projetada para aceitar um Array de Localidades separado por vírgulas. Ex: Oficina Mecânica em Campinas, Valinhos, Vinhedo. O React divide essa string e dispara buscas assíncronas paralelas via Edge Functions no Supabase, trazendo centenas de resultados de todas as cidades aglutinados em uma única tabela instantaneamente.

1.2. Integração Dupla (Google Places API + Serper API)

Não dependemos de apenas uma fonte. O Google Places traz as geolocalizações precisas, Nota de Estrelas (ex: 4.8), Número de Avaliações e o Telefone Oficial. Simultaneamente, a ferramenta aciona a Serper API (Motor de busca orgânica do Google) para cavar links de redes sociais (Instagram, Facebook) e dados corporativos que o Maps costuma esconder. Isso cria um Dossiê ultra-enriquecido de cada Lead.

1.3. O Motor Anti-Concorrência (Blacklist Automática)

Quando os resultados chegam do Google, eles passam por um funil de purificação em Javascript. Nosso código tem um dicionário extenso que bloqueia e oculta da sua visão entidades inúteis para venda de Marketing: Prefeituras, Escolas Públicas, SEBRAE, e o principal: Agências de Marketing (seus concorrentes) e desenvolvedores de software.

Módulo 2: CRM e Classificação da Dor (Scoring)

2.1. O Alerta Supremo: A Tag "SEM SITE"

Esta é a regra de negócio número 1 do sistema. Durante a varredura, o código cruza o places_data gerado. Se a URL oficial do estabelecimento estiver ausente, vazia ou apontando para um provedor morto, a empresa é pintada automaticamente de Vermelho. Nós chamamos isso de DOR MÁXIMA, pois ela fatura no mundo físico, mas não existe no tráfego online.

2.2. Thermal Sorting (Ordenação Térmica)

Sua tabela de leads não é alfabética. Ela flutua os piores casos primeiro.

🔥 Topo 100 Pontos: Empresas Sem Site ou Sem Redes Sociais.
🔥 Meio 50 Pontos: Tem site, mas está caindo aos pedaços (detectado pelo Auditor abaixo).
❄️ Fundo 0 Pontos (Azul): Site perfeito, rápido. O sistema joga pro final pra você não perder tempo vendendo pra quem não precisa.

2.3. O Gerenciamento da Interface (React + Zustand)

A tabela gerencia milhares de contatos sem travar o navegador do usuário porque usa um motor de Paginação e Virtualização Virtual. Seleções em massa permitem adicionar centenas de Leads direto para o Banco do Supabase em um milissegundo, criando filas assíncronas prontas pro robô abater mais tarde.

Módulo 3: O Auditor Técnico Pen-Test

3.1. Hook de Checagem HTTPS (SSL)

Se a empresa TEM um site, você pode apertar o botão "Auditoria". Nosso código TypeScript força conexões de socket de baixo nível para testar certificados. Se o site do cara abrir como "Ir para site Inseguro" pelo Chrome, pegamos isso e montamos a argumentação: "Seu cliente entra no site, o celular diz que tem vírus, ele desiste da compra."

3.2. Integração com Google PageSpeed Mobile

Nós varremos a URL do lead pelo motor Core Web Vitals do Google via API. Se o tempo de carregamento no Mobile (First Contentful Paint) for maior que 2.5 segundos ou o Score geral for menor que 50, anexamos a tag de lentidão. Nós dizemos pra ele que as pessoas clicam, cansam de esperar e fecham a aba.

Módulo 4: Fábrica Invisible de PDF

4.1. Por que NÃO geramos o PDF no seu navegador

Gerar um arquivo gráfico super complexo exige absurdos do Javascript do Chrome do seu computador. Montar 100 PDFs em loop faria sua máquina superaquecer. Para consertar isso, criamos a arquitetura do Motor Node.js Isolado (Porta 3001).

4.2. O Chrome Fantasma e o Puppeteer

O React compila um pacote de dados incrivelmente otimizado com nome da empresa, imagens e as falhas dela cruas. Ele envia isso (POST API) para o nosso projeto backend Node, que dispara a biblioteca Puppeteer. Um Google Chrome Healdess (invisível) acorda na memória RAM do computador, desenha com HTML as réguas das propostas VIP, tira as "fotos da tela", converte isso tudo pra um arquivo formato Adobe PDF real, e joga o texto do arquivo como Base64 puro de volta pro servidor. Nunca salvamos 1 byte de lixo no sistema de arquivos do HD.

Módulo 5: Motor de Disparo Standalone & IA

5.1. Copywriting Instantâneo via Piramyd Cloud (Llama 4)

Quando o robô começa a processar a Fila do Banco Supabase (tabela whatsapp_queue), o primeiro passo é pedir para a inteligência artificial formular o ataque verbal. Usamos a API Piramyd Cloud (por ter latência zero). Os Prompts internos do sistema orientam a IA a agir não como um robô que quer vender, mas como um "Consultor preocupado na região de [Cidade do Lead] que notou que os clientes do [Nicho do Lead] não estão conseguindo achar o [Erro Detectado]". Textos minúsculos, diretos, sem "Prezado Sr/Sra", e 100% orgânicos.

5.2. O Robô Baileys Multi-Device (Fora do Navegador)

Você **não** vai escanear QR Code numa extensão de Chrome praca do Meta que toma Ban com 10 cliques. Nosso projeto utiliza o módulo @whiskeysockets/baileys dentro do ecossistema NodeJS isolado. Ele abre WebSockets oficiais do WhatsApp via protocolo MQTT, simulando ser um celular real. Você lê o QR Code direto no Terminal/Console.

5.3. Defesas Anti-Ban Absurdas do Automação

O Bot nunca chuta às cegas. Segue a lógica blindada:
1️⃣ Checagem nativa de Jid: Verifica silenciosamente com a rede do WhatsApp se o número do Lead tem conta ativa antes de ousar tentar enviar qualquer byte.
2️⃣ Presença Falsa de Atividade (Typing Indicator): O bot liga a tag "Digitando..." e paralisa entre 4 a 12 segundos antes de mandar mensagem, simulando um polegar humano.
3️⃣ O Intervalo Randômico do Caos: O Loop Assíncrono para. Ele sorteia um fator matemático de espera longo (45 a 120 Segundos de Paralisação). Isso achata totalmente a curva de disparos, ficando impossível do algoritmo de Spam do WhatsApp banir seu número de chip, pois você está apenas dialogando no mesmo ritmo de um ser humano respondendo várias pessoas de manhã.

5.4. Interface WebSocket em Tempo Real Mágica

Enquanto isso, voltando para a interface bonita em React no seu navegador, a tela de Automação está completamente atualizada sozinha. Assim que o robô Node processa o envio com sucesso no terminal, ele altera a linha no banco Supabase para status: 'sent'. Se conectamos um Subscribe via WebSockets nativo no banco. A sua tela do navegador reage no mesmo piscar de olhos mudando os crachás de estado e empurrando barrinhas gráficas circulares de "Progresso %". Você monitora mil envios bebendo um café na rua enquanto o PC em casa esmaga a fila.
